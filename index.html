Hexo+nexT疑难杂症

发表于 2015-10-13   |   分类于 Hexo

在尝试使用Hexo（3.0）搭配nexT主题时出现了各种问题，折腾很久终于解决了部分问题，总结如下。

Hexo+nexT的搭建

对于搭建Hexo环境和配置nexT主题，网上有很多教程，这里推荐Joe’s Room的博客：
在GitHub上搭建基于Hexo的静态Blog
按照博主的步骤一步一步亲测可以搭建成功。

Hexo 3.0 Deploy出错 Not Found Git

Hexo 3.0需要修改站点配置文件中的_config.yml文件，对于文件中的deploy字段，应该将type由github修改为git

npm install hexo-deployer-git –save

设置当前 page 类型为 tags

type: “tags”

设置当前页评论关闭

comments: false

```

以上述tags为例，如果你是使用hexo new page tags来新建的tags文件夹，会有一个问题就是，在上述md文件的字段中会默认有title和date两个字段（当然你可以修改scaffolds中的文件默认值），而默认生成的date字段的格式是：YYYY-MM-DD HH：MM：SS类似这种类型的，而这种格式很可能就不会被识别，所以我推荐对于tags和categories文件直接将title和date字段删除掉，并无大碍。亲测成功！

未完待续。。。
